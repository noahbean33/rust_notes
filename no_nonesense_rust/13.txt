Lesson 11 - Performance Optimization

This experiment shows you exactly how to write fast Rust code. And also, it shows you which kinds of optimizations don't bring any value. After this lesson you'll know exactly how to write fast programs in Rust.

THE #1 RULE: cargo build --release

In Rust, the single biggest performance lever is building in release mode. Debug builds are intentionally slow (no optimizations, extra checks). Release builds use full LLVM optimizations — the same backend as clang.

cargo build                  # debug mode — SLOW (good for debugging)
cargo build --release        # release mode — FAST (good for benchmarks and production)

Typical speedup: 10x to 50x. Always benchmark with --release.

Measuring execution time:

time cargo run --release
time ./target/release/your-program

Exercise

Optimize the following source code. First, make sure you're running in release mode. Then measure its execution time before and after your changes using time. How much time can you shave off?

struct Vec3 {
    x: f64,
    y: f64,
    z: f64,
}

fn vec_length(v: &Vec3) -> f64 {
    v.x * v.x + v.y * v.y + v.z * v.z
}

fn vec_add(a: &Vec3, b: &Vec3) -> Vec3 {
    Vec3 {
        x: a.x + b.x,
        y: a.y + b.y,
        z: a.z + b.z,
    }
}

fn vec_scale(v: &Vec3, s: f64) -> Vec3 {
    Vec3 {
        x: v.x * s,
        y: v.y * s,
        z: v.z * s,
    }
}

fn main() {
    let mut position = Vec3 { x: 1.0, y: 2.0, z: 3.0 };
    let velocity = Vec3 { x: 0.1, y: 0.2, z: 0.3 };

    for _ in 0..1_000_000 {
        position = vec_add(&position, &velocity);

        if vec_length(&position) > 1000.0 {
            position = vec_scale(&position, 0.5);
        }
    }

    println!("Final position: {:.2} {:.2} {:.2}",
             position.x, position.y, position.z);
}

slow.rs — debug build, no optimizations

fn square(n: i32) -> i32 {
    n * n
}

fn main() {
    let mut n: i32 = 1;
    for _ in 0..3_000_000_000u64 {
        let square_result = square(n);
        n = n.wrapping_add(square_result);
    }
    println!("Value: {}", n);
}

Build and time this two ways:

cargo build
time ./target/debug/slow

cargo build --release
time ./target/release/slow

The difference is massive. In C, you had to manually pass-by-pointer and hoist variables to get speed. In Rust, the LLVM optimizer does all of that for you in release mode. Your job is to write clear code and let --release handle the rest.

Key Rust performance insights:

1. --release is the #1 optimization. Never benchmark debug builds.
2. Small types (i32, f64) are Copy — passing by value is as fast as passing by reference. The optimizer inlines everything.
3. Iterators are zero-cost abstractions — they compile to the same assembly as hand-written loops.
4. Use #[inline] hints sparingly — the compiler usually makes good inlining decisions already.
5. For real benchmarks, use the criterion crate: cargo add criterion --dev
